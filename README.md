# ContentSphere — Generate Your Words ✨

An AI-powered content generation chat interface built with React and Node.js. Users can generate emails, blog posts, social media posts, and summaries through a conversational UI backed by Google Gemini AI — with a two-layer caching system (Redis → MySQL) to reduce API calls and response times.

---

## Features

- **4 content modes** — Email, Blog Post, Social Post, Summary — each with tailored prompts
- **Conversational chat UI** — message history, typing indicator, timestamps
- **Smart caching** — Redis (fast in-memory) → MySQL (persistent) → Gemini API fallback
- **Source badge** — every response shows where it came from (⚡ Redis, 🗄️ MySQL, or 🤖 Gemini)
- **Copy to clipboard** — hover any bot message to copy with one click
- **Dark mode** — full light/dark theme toggle
- **Responsive** — mobile sidebar with overlay, desktop persistent sidebar
- **Mode tabs** — color-coded content type selector in the input area

## Getting Started

### Prerequisites

- Node.js v18+
- Redis (running locally or via Docker)
- MySQL
- Google Gemini API key

### 1. Clone the repository

```bash
git clone https://github.com/prashant0413/ContentSphere-Generate-you-words.git
cd ContentSphere-Generate-you-words
```

### 2. Backend setup

```bash
cd Backend
npm install
```

Create a `.env` file in the `Backend` folder:

```env
PORT=5000
GEMINI_API_KEY=your_gemini_api_key_here

# MySQL
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=contentsphere

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
```

Start the backend:

```bash
npm start
```

The API will run at `http://localhost:5000`.

### 3. Frontend setup

```bash
cd Frontend
npm install
npm run dev
```

The app will run at `http://localhost:5173`.

## API Reference

### `POST /api/content/generate`

Generates content based on type and prompt.

**Request body:**
```json
{
  "type": "email",
  "prompt": "Write an email asking my manager for a day off"
}
```

**Response:**
```json
{
  "content": "Subject: Leave Request...",
  "source": "gemini_api"
}
```

**Content types:** `email`, `blog`, `social`, `summary`, `general`

**Source values:**
- `redis_cache` — served from Redis (fastest)
- `mysql_cache` — served from MySQL (cached in DB)
- `gemini_api` — freshly generated by Gemini AI

---

## Caching Flow

```
Request → Check Redis → Hit? Return instantly (⚡)
               ↓ Miss
          Check MySQL → Hit? Return + repopulate Redis (🗄️)
               ↓ Miss
          Call Gemini API → Store in MySQL + Redis → Return (🤖)
```

This ensures repeated prompts are served instantly without burning API quota.
